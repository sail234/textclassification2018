。 产生 于 年代 中期 的 CSCW 技术 与 教育 的 特点 不谋而合 。 将 CSCW 技术 与 CAI 相结合 ， 就 产生 了 协同式 学习 ， 即 老师 和 多个 学生 能 对 某 一 问题 进行 分析 、 讨论 、 交流 等 协作 活动 ， 从而 深化 对 问题 的 理解 ， 获得 高级 的 认知 能力 。 　 　 CSCW 系统 须 采用 符合 群体 工作 方式 特点 的 技术 ， 多 Agent 系统 适用 于 CSCW 系统 。 Agent 是 指 在 一定 环境 下 自主 运行 ， 包含 了 信念 、 承诺 、 义务 、 意图 等 精神状态 的 实体 。 其 特点 是 ： 学习 性 、 知识性 、 自主性 和 相互合作 性 。 在 多 Agent 系统 的 学习 中 ， Agent 是 平等 协作 的 ， 以 协同 操作 来 共同完成 给定 的 学习 任务 ， 其 学习 特性 与 现代 教育 相吻合 。 此外 ， Agent 的 智能性 更好 地 体现 了 个体 在 协同工作 中 的 重要 作用 。 所以 ， 多 Agent 系统 能 很 好 地 实现 CSCW 系统 ， 为 CSCW 系统 提供 良好 的 系统 解决方案 ， 结合 使用 Agent 技术 已 成为 协同 CAI 领域 的 一个 重要 趋势 。 　 MAC 模型 结构 　 　 基于 MultiAgent 和 CSCW 的 观点 ， 以 现代 的 网络 技术 、 超媒体 技术 为 基础 ， 采用 面向对象 的 设计 方法 ， 提出 了 一个 具有 客户服务器 模式 的 多媒体教学 模型 MAC 模型 。 介绍 MAC 模型 之前 ， 先 介绍 该 模型 中 Agent 和 CSCW 的 应用 情况 。 　 Agent 类型 的 确定 　 　 MultiAgent 系统 是 由 一组 彼此间 进行 协作 或 竞争 的 主体 组成 的 系统 。 在 涉及 到 Agent 的 系统 中 ， 首先 要 根据 问题 确定 系统 中所 需 关注 的 智能 主体 。 角色 机制 是 一个 协同 系统 正常 运作 所 不可 缺少 的 ， 角色 是 一组 提供 给 协同 教学 的 权利 和 责任 的 属性 。 在 本 协同 教学 模型 中 显然 有 两种 角色 ： 学生 和 教师 。 根据 这 两种 角色 可 相应 确定 两类 Agent ： 用以 辅助 教师 对 教学 进行 某些 管理 和 控制 的 教师 Agent 和 用以 辅助 学生 进行 学习 的 学生 Agent 。 在 MAC 系统 中 ， 为了 更好 地 协调 位于 server 端的 教师 Agent 与 位于 client 端的 学生 Agent 之间 的 协作 活动 ， 每类 Agent 包括 若干个 具有 特定 功能 的 应用 子 模块 。 学生 Agent 包括 两 部分 应用 子 模块 ： 一部分 用于 支持 学生 的 学习 活动 ， 另 一部分 用于 支持 学生 对 当前 的 学习 状况 进行 自我 评测 ， 主要 是 通过 与 评测 库 的 交互 来 确定 学生 的 知识 水平 和 错误 所在 。 教师 Agent 包括 部分 应用 子 模块 ： 第一 部分 用于 监控 学生 的 学习 进度 、 当前 学习 状态 ， 以 确定 对 该生 应用 何种 导航 策略 ， 并 监控 共享资源 的 使用 情况 ； 第二 部分 通过 与 协同 规则 库 的 交互 来 解释 显式 定义 的 协同 协议 处理 协同 要求 ； 第三 部分 利用 监控 得到 的 信息 和 评测 库对 全体学生 的 总体 学习 状况 加以 评测 ， 以 改进 教学方法 ， 改善 教学效果 ； 第四 部分 用于 对 学生 进行 指导 ， 进行 交流 、 探讨 。 　 协作 教学 的 类型 　 　 CSCW 系统 的 目的 就是 在 计算机 环境 下 提供 对 人们 群体 工作 的 支持 。 一般 的 协同 系统 中 ， 按照 交互 合作 方式 和 协同 者 的 地域 分布 而 从 时间 和 空间 的 概念 来 界定 ， 大致 有 以下 种 协同 方式 ： 同步 模式 、 分布式 同步 模式 、 异步 模式 、 分布式 异步 模式 。 　 　 MAC 系统 作为 一个 协作 教学系统 ， 扩充 了 原有 CAI 的 教学模式 ， 主要 在于 支持 协同工作 。 MAC 系统 支持 种 协同工作 ： 个别辅导 、 群体 指导 和 自由 讨论 。 个别辅导 是 由 学生 通过 学生 Agent 向 教师 Agent 发出请求 ， 请求 到达 教师 Agent 请求 队列 该 队列 采用 先来 先 服务 的 策略 后 ， 等待 调度 ， 一旦 该 请求 调度 成功 ， 则 学生 和 教师 之间 进行 一对一 的 交流 。 与 此 不同 的 是 ， 当 教师 Agent 发现 个别辅导 请求 队列 中有 许多 请求 相近 ， 或者 当 教师 Agent 启动 监控 功能 发现 知识库 中 某 一 知识点 多次 被 确定 为 错误 所在 ， 教师 Agent 就 撤消 所有 同类 的 个别辅导 的 请求 ， 改为 一个 群体 指导 的 请求 。 群体 指导 的 优先权 高于 个体 辅导 ， 一旦 群体 指导 调度 成功 ， 教师 与 学生 之间 进行 一对 多 的 交流 。 自由 讨论 则 是 由 教师 Agent 自己 将 请求 加入 到 协同 请求 队列 中 等待 调度 ， 以 对 某些 教师 认为 值得 讨论 的 问题 进行 自由 讨论 。 　 MAC 模型 　 　 综合 考虑 确定 的 Agent 类型 和 协作 类型 ， 提出 MAC 系统 的 设计 模型 如图所示 。 该 模型 构造 了 一个 合适 的 多媒体教学 网络 的 体系结构 ， 建立 了 师生 协同 教学 的 模型 ， 将 传统 的 CAI 教学 推进 到 一个 以 超媒体 为 基础 ， 地理分布 协同工作 的 网络 教学 层面 。 图 　 MAC 系统 设计 模型 图 　 MAC 系统 部分 的 关键技术 　 Agent 的 具体 表示 　 　 MAC 系统 共有 类 Agent 。 从 形式化 的 角度 来看 ， Agent 可以 用 一个 六元 组来 描述 ： 　 　 ADepiction ＝ ＜ Aid ， F ， S ， T ， R ， C ， 其中 Aid 是 Agent 的 标识符 ， F 是 Agent 的 外部 征 ， S 　 是 Agent 的 状态 集 ， T 是 Agent 的 事务处理 集 ， R 是 Agent 的 资源 集 ， C 是 定义 在 T 和 R 上 的 约束 。 结合 面向对象 技术 ， Agent 可以 用 封装 特殊 内部结构 的 类来 表示 。 因此 ， MAC 系统 中 的 Agent 对象 是 一个 五元 组 AObject ＝ ＜ Aid ， A ， M ， C ， I ， 其中 ， Aid 是 AObject 的 唯一 标识符 ， 与 ADepiction 中 的 Aid 相对 应 ； A 是 AObject 具有 一定 数据结构 的 私有 存储 区域 ， 为 属性 集 ， 与 ADepiction 中 的 F 、 S 和 R 相对 应 ， 如 Agent 对象 类型 ； M 是 AObject 的 操作 集合 ， 定义 了 该类 对象 为 完成 其 任务 而 必须 具有 的 自主 行为 特性 ； C 是 AObject 的 协作 操作 集 ， 定义 了 当 条件 符合 时 ， 该类 AObject 触发 协同 请求 时 或 该类 AObject 被 请求 协同工作 时 的 操作 ； I 是 AObject 与 网络 和 其它 AObject 的 接口 操作 。 这个 部分 共同 组成 ADepiction 中 的 T 事务处理 集 。 　 MAC 系统 Agent 的 通信 　 　 由于 MAC 是 一个 基于 MultiAgent 和 CSCW 的 多媒体教学 系统 ， 而 一个 协同 系统 中 ， 通信 是 基础 ， 合作 是 形式 ， 协调 是 关键 。 与 此 相对 应 的 是 ： 从 功能 的 角度 来看 ， Agent 的 表达 结构 可 分为 层 ： 通信 层 、 自定义 层 和 管理层 。 其中 通信 层 包括 网络接口 和 通信接口 ， 负责 完成 Agent 实体 之间 的 交互 和 通信 ， 与 中 AObject 的 I 部分 相对 应 。 自定义 层 定义 了 Agent 的 类型 、 内部结构 及其 承担 的 任务 ， 对应 于 AObject 中 的 Aid ， A ， M 部分 。 管理层 则 定义 了 协作 协议 ， 并且 通过 通信 层 传递 过来 的 消息 以 启动 Agent 中 相应 的 应用 子 模块 ， 主要 由 AObject 中 的 C 部分 来 实现 。 在 多 Agent 系统 中 ， Agent 的 协作 是 通过 Agent 之间 的 交互 来 实现 的 。 任何 一种 交互方式 的 具体 实现 都 是 通过 通信 层 实现 的 。 所以 通信 层 的 实现 是 多 Agent 系统 中 的 一个 关键问题 。 Agent 之间 的 通信 一般 有种 方式 ： 第一种 是 私线 privateline 方式 ， 即 Agent 之间 进行 保密性 良好 的 直接 通信 ； 第二种 方式 是 黑板 blackboard 方式 ， 即 把 信息 放在 广泛 可取 的 黑板 上 ， 实现 广播 通信 ； 第三种 方式 是 预定 点 方式 ， 通信 的 Agent 将 信息 放在 一个 约定 好 的 位置 ， 实现 间接 通信 ； 第四种 方式 介于 二 和 三 之间 ， Agent 进行 分组 ， 每个 组都 有 相关 的 黑板 。 在 MAC 系统 中 ， Agent 之间 的 通信 方式 有 多种 ： 个别辅导 时 采用 私线 方式 ， 而 群体 指导 以及 自由 讨论 时 采用 黑板 方式 。 考虑 到 KQMLKnowledgeQueryandManipulateLanguage 现已 成为 定义 Agent 之间 通信 语法 和 语义 规范 的 主体 通信 语言 的 事实标准 ， 设计 了 与 KQML 语言 兼容 ， 并且 适合 CBuilder 编程 环境 下 使用 的 特定 Agent 通信 语言 ， 该 语言 可 形式化 地 表示 成为 一个 四元组 ： 动作 ， 源 Agent ， 目标 Agent ， 内容 。 　 　 此外 ， 系统 采用 TCPIP 协议 的 ClientServer 模式 。 通常 TCPIP 协议 只 提供 点到点 的 服务 ， 这是 一种 面向 服务 的 协议 ， 系统 大量 需要 教师 Agent 和 学生 Agent 之间 一对 多 的 交互 。 为了 通过 网络 编程 套 接字 Socket 来 实现 多点 传输 ， 系统 采用 主从 方式 ， 通过 教师 Agent 软件 模拟 多点 传输方式 ， 每个 参与 多点 传输 的 学生 Agent 以及 教师 Agent 都 有 一个 成员 状态栏 ， 且 学生 Agent 只 需 与 教师 Agent 进行 连接 ， 教师 Agent 启动 监控 功能模块 检测 成员 的 状态 ， 并 动态 维护 成员 状态栏 ， 以此 为 教师 Agent 的 广播 功能 提供 支持 ， 各 学生 Agent 进行 操作 时 ， 只 需向 教师 Agent 发送数据 ， 教师 Agent 根据 成员 状态栏 中 的 信息 转发 信息 包 。 　 MAC 系统 Agent 工作 区 　 　 由于 学生 和 教师 这 两种 角色 权利 不同 ， 其 工作 区 的 人机界面 尽管 形式 上 统一 ， 但 其 操作 不尽相同 。 学生 Agent 和 教师 Agent 都 有 以下 个 部分 工作 空间 ： 个人 工作 区 、 布告栏 、 自由 讨论区 和 成员 状态栏 。 显而易见 个人 工作 区 只 供 Agent 本身 使用 ， 是 每个 Agent 私有 的 。 而 布告栏 则 是 用于 教师 进行 群体 指导 或 发布 重要 通知 。 因此 ， 教师 Agent 对 布告栏 拥有 读 、 写权 ， 而 学生 Agent 就 只有 浏览 权 。 自由 讨论区 用于 自由 讨论 ， 这部分 界面 需要 实现 CSCW 对 界面 环境 的 一个 基本 要求 是 ： 你 见 即 我 见 WYSIWIS ： WhatYouSeeIsWhatISee 。 所有 加入 自由 讨论区 的 Agent 都 有 使用 和 浏览 的 权利 ， 所以 这里 有 一个 保证 自由 讨论区 窗口 内容 一致性 的 问题 。 采用 发言权 控制 来 解决 这一 问题 ， 即 在 任一 时刻 ， 至多 只有 一个 Agent 拥有 发言权 ， 即 对 自由 讨论区 拥有 使用权 。 发言权 调度 采用 教师 控制 加 排队 机制 ， 排队 机制 按照 先进先出 加 优先权 调度 的 方式 进行 ， 一般 先 请求 发言 的 Agent 拥有 高 优先权 ， 但 教师 可以 动态 地 改变 每个 学生 的 优先权 ， 而且 教师 Agent 的 优先权 任何 时候 都 是 最高 的 ， 且 该 调度 是 抢占 式 的 ， 即 一旦 出现 高 优先权 请求 发言 ， 立即 抢占 正在 发言 的 Agent 的 发言权 ， 发言权 的 切换 由 教师 Agent 启动 协同 功能模块 来 调度 。 而 成员 状态栏 的 使用者 是 教师 Agent ， 学生 Agent 都 只有 浏览 权 ， 除了 权利 不同 之外 ， 学生 Agent 只有 加入 自由 讨论区 后 ， 才 启动 其 成员 状态栏 ， 显示 内容 为 当前 发言权 拥有者 以及 参加 自由 讨论 各 成员 的 当前 状态 ， 而 教师 Agent 一旦 启动 ， 其 成员 状态 区 随即 启动 ， 显示 内容 为 当前 登录 的 所有 学生 的 当前 状态 、 学习 进度 等 信息 。 　 其它 　 　 利用 Winsock 编程 ， MAC 系统 还 提供 了 email 等 工具 以 方便 师生 之间 进行 交流 。 Winsock 是 基于 Socket 模型 和 Windows 消息 驱动 机制 的 Windows 扩展 函数 。 Winsock 支持 两种 套 接字 ： 流式 套 接字 和 数据包 式 套 接字 。 本 模型 对 通信 服务 的 可靠性 要求 较 高 ， 因而 采用 流式 套 接字 。 　 　 协同 教学 中 的 课件 库 必须 具有 功能强大 、 结构合理 的 信息 模型 。 由于 超媒体 容纳 了 多种 成份 节点 、 链 ， 因此 系统 的 课件 库 等 采用 基于 超媒体 文档 的 结构 ， 以 满足 协同 访问 、 表达 出 教学 知识 的 内在联系 的 要求 。 　 结束语 　 　 基于 MutiAgent 和 CSCW 的 多媒体教学 系统 支持 教师 和 学生 之间 以及 学生 之间 的 协同工作 和 信息 交流 过程 ， 不但 为 学生 提供 了 一个 生动活泼 、 社会化 和 个人化 并存 的 学习 环境 ， 而且 大大 地 提高 了 学生 和 教师 在 完成 教学 工作 任务 过程 中 相互合作 和 信息 交换 的 质量 ， 适应 了 信息化 社会 对 教育 的 需要 。
