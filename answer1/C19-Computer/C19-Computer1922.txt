微型机 与 应用 MICROCOMPUTERITSAPPLICATIONS 　 Vol 　 No 　 P 用 VB 创建 支持 断点续传 的 Internet 传输 控件 凌小 鹏 　 曹迪铭 摘要 ： FTP 的 功能 、 实现 方法 及 利用 FTP 实现 断点续传 的 原理 ， 结合 VisualBasic 编程 讲述 了 创建 支持 断点续传 的 Internet 传输 控件 的 过程 。 关键词 ： 断点续传 MicrosoftInternetTransfer 控件 ActiveX 控件 VisualBasic 编程 　 　 随着 Internet 的 普及 ， FTP 以其 进行 文件传输 便捷 、 通用 的 特点 ， 正 被 越来越 频繁 地 使用 。 虽然 已有 很多 实用 的 现成 程序 可用 ， 但 许多 用户 还是 希望 自己 设计 和 定制 具有 FTP 功能 的 应用 。 而 VB 的 MicrosoftInternetTransfer 控件 就是 为 编程 者 广为 使用 的 工具 之一 ， 用户 以此 为 基础 ， 可 开发 出 自己 的 支持 文件传输 的 应用 。 但 VB 本身 却 并 不 含有 支持 断点续传 的 控件 ， 因此 这里 设计 了 支持 断点续传 的 Internet 传输 控件 ， 供 编程人员 参考 使用 。 FTP 简述 及 断点续传 的 原理 　 　 FTP 文件传输 协议 是 Internet 上 使用 最 广泛 的 协议 之一 。 使用 FTP 协议 可以 在 FTP 服务器 上 登录 ， 实现 文件 的 上 、 下载 。 FTP 本身 是 一个 具有 客户 ／ 服务器 结构 的 系统 ， 软件系统 由 客户机 和 服务器 上 的 部分 组成 。 在 FTP 的 协议 内部 定义 了 一组 可 供 用户 使用 的 命令 用以 完成 文件 的 传送 ， 如 ： 切换 目录 的 命令 CD 、 获取 文件 的 命令 GET 及 上载 文件 的 PUT 命令 等 。 文件传输 的 实现 实际上 就是 客户机 和 服务器 上 的 软件 依据 FTP 协议 相互 通信 发出请求 命令 和 响应 的 结果 。 其中 ， 在 私有 的 FTP 服务器上进行 登录 还 需 验证 用户 身份 。 　 　 FTP 是 由 一组 命令 实现 的 ， 基本 的 FTP 命令 如下 ： 　 　 CDfile 改变 目录 ： 改变 到 file 中 指定 的 目录 。 　 　 CDUP 改变 到父 目录 ： 等效于 “ CD ” 。 　 　 CLOSE 关闭 当前 的 FTP 连接 。 　 　 DELETEfile 删除 file 中 指定 的 文件 。 　 　 DIRfile 目录 ： 搜索 file 中 指定 的 目录 （ 允许 用通 　 　 　 　 配符 ， 但 要 使用 远程 主机 的 语法 。 ） ， 如果 没 　 　 　 　 有 指定 file ， 将 返回 当前 的 整个 工作 目录 。 　 　 GETfilefile 检索 file 中 指定 的 远程 文件 ， 并 　 　 　 　 　 　 　 　 　 　 创建 file 中 指定 的 新 本地 文件 。 　 　 LSfile 列表 ： 搜索 file 中 指定 的 目录 （ 允许 用通 　 　 　 　 　 　 配符 ， 但 要 使用 远程 主机 的 语法 ） 。 　 　 MKDIRfile 创建 目录 ： 创建 file 中 指定 的 目录 ， 　 　 　 　 　 　 　 　 创建 是否 成功 取决于 用户 在 远程 主机 上 的 权限 。 　 　 PUTfilefile 复制 file 指定 的 本地 文件 到 file 　 　 　 　 　 　 　 　 　 　 指定 的 远程 主机 上 。 　 　 PWD 打印 工作 目录 ： 返回 当前目录 名 。 　 　 QUIT 终止 当前 用户 。 　 　 RECVfilefile 　 检索 file 中 指定 的 远程 文件 ， 　 　 　 　 　 　 　 　 　 　 　 并 创建 file 中 指定 的 本地 新 文件 ， 等效于 GET 。 　 　 RENAMEfilefile 　 将 file 中 命名 的 远程 文件 　 　 　 　 　 　 　 　 　 　 　 　 重命名 为 file 中 指定 的 新 名称 ， 成功 　 　 　 　 　 　 　 　 　 　 　 　 与否 取决于 用户 在 远程 主机 上 的 权限 。 　 　 RMDIRfile 　 删除 目录 ： 删除 file 中 指定 的 远程 　 　 　 　 　 　 　 　 目录 ， 成功 与否 取决于 用户 在 远程 主机 上 的 权限 。 　 　 SENDfilefile 　 复制 file 指定 的 本地 文件 到 　 　 　 　 　 　 　 　 file 指定 的 远程 主机 上 ， 等效于 PUT 。 　 　 SIZEfile 返回 file 指定 的 目录 大小 。 　 　 这些 命令 中 ， 用来 从 远程 主机 上 获取 文件 的 最 常用 的 命令 是 GET 。 另外 ， 还有 个 有用 的 FTP 命令 并未 被 列入 ， 它们 是 RESTART 和 REGET 。 　 　 RESTART 的 功能 是 在 用户 使用 GET 命令 从 远程 主机 上 下载 文件 ， 或 使用 PUT 命令 向 远程 主机 上载 文件 中途 出现 了 线路 问题 ， 而 引起 传输 过程 中断 时 运行 RESTART 命令 ， 系统 会 在 下 一个 GET 命令 或 PUT 命令 运行 时 ， 让 其 从 上次 被 中断 处 重新 开始 上 、 下载 ， 从而 使 用户 不必 从头开始 ， 节约 传输 文件 的 时间 。 　 　 REGET 的 功能 是 若 用户 在 使用 GET 命令 从 远程 主机 下载 文件 的 过程 中 发生 中断 时 ， 使用 REGET 命令 可以 实现 从 中断 处 重新 开始 文件传输 。 　 　 以上 个 命令 在 线路 质量 不高 、 传输速率 不 稳定 或 传输 的 文件 较大 时 ， 显得 十分 有用 ， 可以 借助于 它们 设计 出 支持 断点续传 的 应用 。 需要 注意 的 是 ， 由于 FTP 所处 的 环境 和 实现 的 软件 相差 很大 ， 各个 FTP 系统 支持 的 命令集 也 不尽相同 。 上述 的 RESTART 和 REGET 命令 的 实现 将 有赖于 系统 本身 的 支持 ， 即 断点续传 能否 实现 还要 看 对方 服务器 是否 支持 。 否则 ， 即便 采用 了 上述 方法 也 不 能够 实现 断点续传 。 支持 断点续传 的 控件 制作 控件 功能 的 规划 　 　 个 支持 断点 的 控件 应 包括 以下 的 基本功能 ： 　 　 （ ） 方法 ： START 、 CANCEL 、 PAUSE 方法 ， 其中 START 表示 开始 传输 或 中断 后 继续 传输 ； CANCEL 表示 中断 传输 ； PAUSE 表示 暂停 传输 过程 ， 并 可以 用 START 再次 恢复 传输 。 　 　 （ ） 属性 ： URL 属性 ， 表示 远程 主机 的 位置 ； FPOS 属性 ， 表示 文件 在 远程 主机 上 的 目录 位置 ； DIR 属性 ， 表示 下载 文件 保存 在 本地 机上 的 路径 和 文件名 ； RunCode 属性 ， 表示 正在 进行 文件传输 的 状态 。 为 Inet 。 　 　 （ ） 在 VB 的 “ 外接 程序 ” 菜单中选择 “ 外接 程序 管理器 ” 打开 “ 外接 程序 管理器 ” 窗口 ， 在 “ 可用 外接 程序 ” 列表 中 选择 “ VBActiveX 控件 接口 向导 ” ， 同时 在 窗口 右下角 将 “ 在 启动 中 加载 ” 和 “ 加载 ／ 卸载 ” 的 复选 标记 选中 ， 并 按 “ 确定 ” 。 　 　 （ ）
