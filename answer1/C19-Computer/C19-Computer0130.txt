Keywords 　 ASPIISDHTMLADO 　 前言 　 　 随着 Internet 应用 的 不断 扩展 许多 企业 内部 都 建有 Intranet 。 加强 企业 内部 信息 的 集成 与 共享 ， 通过 Intranet 及时 获取 企业 内部 的 综合信息 ， 对于 改善 和 优化 企业 管理 、 增强 企业 整体效益 将 提供 有力 的 支持 。 为此 ， 我们 设计 和 开发 了 一个 基于 浏览器 服务器 方式 的 Web 数据库 综合 查询 系统 。 该 系统 是 一个 通用 的 决策 支持 综合 查询 系统 ， 由于 基于 Web 、 ODBC 、 TCPIP 等 技术 ， 实现 了 异构 数据库 和 异构 网络平台 的 数据 综合 查询 。 该 系统 采用 ASP 开发 环境 ， 为此 有 必要 先对 ASP 做 一个 简要 的 介绍 。 　 ASP 简介 　 　 ASP （ ActiveServerPages ） 是 微软 推出 的 服务器端 脚本 编写 环境 ， 使用 它 可以 组合 HTML 、 脚本 命令 和 ActiveX 组件 ， 创建 和 运行 动态 、 交互 的 基于 Web 的 应用程序 。 ASP 具有 以下 特点 ： 　 　 开发 、 维护 快速 简单 。 ASP 文件 可 通过 通用 的 文本编辑 器 （ 如 记事本 ） 或 专门 的 集成 开发 环境 VisualInterDev 进行 开发 　 　 编程语言 通用 。 ASP 可 使用 VBScript 、 Javascript 、 Perl 等 多种 脚本语言 进行 编程 　 　 内置 对象 和 组件 。 为了 方便 编程 ， ASP 除了 提供 多种 功能强大 的 内置 对象 和 组件 ， 还 允许 用户 编写 自己 定制 的 ActiveX 控件 　 　 安全性 。 ASP 使用 NT 的 安全 机制 ， 通过 NT 授权 机制 可以 规定 使用 范围 　 　 与 浏览器 无关 。 ASP 是 服务器端 的 脚本 执行 环境 ， 只 需 将 脚本 的 执行 结果 送给 客户端 浏览器 。 由于 执行 结果 一般 只 包含 常规 超文本 标识 ， 所以 不 依赖 客户端 的 浏览器 。 　 　 Web 数据库 综合 查询 系统对 运行 环境 有 一定 的 要求 ， 但是 这些 要求 比较 容易 满足 。 运行 环境 主要 包括 服务器端 和 客户端 两 方面 。 Web 服务器端 要求 操作系统 采用 WindowsNT ， 必须 经过 ServicePack 升级 ， 安装 了 NTOptionPack ， Web 服务器软件 采用 IIS ； 客户端 操作系统 可以 是 Windows 或 WindowsNTWorkstation ， 如果 不 使用 DHTML ， 可以 是 IE 或 其他 浏览器 （ 如 ： Navigator ） ， 否则 必须 使用 IE 。 数据库系统 无 特殊要求 ， 只须 支持 ODBC 即可 ， 并且 数据库系统 可 安装 于 Web 服务器 上 ， 也 可 安装 于 另外 服务器 上 ， 成为 数据库 服务器 。 　 系统 设计方案 　 　 本 系统 的 设计 目标 是 在 Intranet 基础 之上 ， 以 浏览器 服务器 体系结构 作为 基本 架构 ， 通过 ASP 方式 实现 交互式 、 动态 的 Web 综合 查询 系统 。 基本 设计 思路 是 ， 客户端 只 需 浏览器 并且 可 访问 Web 服务器 ， 服务器端 具体 完成 客户 选择 的 查询 功能 ， 并且 返回 结果 给 客户 。 如此 反复 ， 客户端 就 能 动态 的 获取 数据库 中 的 信息 。 　 　 要 完成 对 数据库 的 查询 ， 就 必须 有 相应 的 完成 所 需 特定 功能 的 SQL 语句 ， 这些 SQL 语句 需要 事先 设计 好 ， 存放 于 数据库 的 相应 的 表中 ， 并且 还要 保存 它们 的 分类 信息 ， 方便 用户 的 检索 和 使用 。 系统结构 框架 如图所示 。 图 　 系统结构 框架 　 　 图中 的 箭头 表示 系统 中 数据 的 流向 。 其中 ， 从 数据库 流向 Web 服务器 的 数据 ， 包括 在 ASP 脚本 中 执行 的 完成 特定 查询 功能 的 SQL 语句 的 信息 ， 和 作为 SQL 语句 查询 对象 的 企业 的 综合信息 。 　 　 系统 的 内部结构 分为 如图所示 的 两层 ： 固定 信息 层 和 动态 查询 层 。 固定 信息 层以 超文本 语言 文件 的 形式 存储系统 的 静态 信息 ， 一般 不会 改变 。 动态 查询 层 由 许多 完成 特定 数据 查询 的 功能模块 组成 ， 一个 模块 就是 一个 后缀 为 asp 的 文件 ， 模块 之间 可以 通过 包含 语句 交叉 引用 。 模块 中 可能 包含 许多 用 脚本语言 书写 的 过程 ， 共同 实现 其 特定 功能 。 图 　 系统 内部 结构图 　 系统 实现 　 　 系统 实现 主要 分为 两 部分 ： SQL 生成 和 Web 查询 。 首先 要 实现 SQL 语句 生成 的 功能模块 ， 这 一部分 并 不 要求 也 采用 Web 方式 实现 ， 采用 客户服务器 的 方式 开发 即可 。 系统 界面 如图所示 。 图 　 SQL 生成 的 界面 　 　 为了 用户 使用方便 ， 本 系统 通过 读取 数据库 的 系统 表 信息 ， 以 可视化 方式 显示 数据库 中 的 表和字 段 的 属性 ， 用户 进行 相应 的 选择 就 可以 快速
